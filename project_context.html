POGLS Storage & Streaming System

Status: Planning | Date: 2026-04-07

Overview

A file and model weights storage system with compression and streaming capabilities designed for LLM inference. Converts files to coordinate sets for efficient storage and enables partial reconstruction for streaming use cases.

Pipeline Flow

File / Model weights ↓ [Section 1: File Scanner] → chunk → coordinate ↓ [Section 2: Compression Layer] ↓ [Section 3: POGLS Storage] ← Existing system ↓ [Section 4: Reconstruct + Stream] ↓ [Section 5: LLM Streaming]

Section Breakdown

Section 1: File Scanner

Converts files to coordinate sets instead of storing content directly.

FileRole
pogls_scanner.hScan file → chunk → coordinate
pogls_file_index.hStore metadata (name, size, type, chunk map)

Rules: Skip compressed files (.zip, .jpg, .mp4), chunk size = 64B (DiamondBlock), index stored separately from data.

Section 2: Compression Layer

Reduces coordinate data size while maintaining reconstructability.

FileRole
pogls_compress.hCompressor / decompressor
pogls_chunk_index.hRandom access index

Strategies: Delta encode, PHI symbol table, Run-length encode, QRPN guard per chunk.

Target

  • Structured data (text, code, weights) → 60-90% reduction
  • Compressed data (.zip, .jpg) → Skip, store as-is

Section 3: POGLS Storage

EXISTING - Do not modify

Uses: pogls_compress.h → pogls_delta (store), FiftyFourBridge (index), Federation (commit), libpogls_v4.so (API)

Section 4: Reconstruct + Stream

Retrieve data completely or partially for streaming use cases.

FileRole
pogls_reconstruct.hReconstruct pipeline
pogls_stream.hPartial / streaming output

Partial reconstruct: Video/audio playback before full load, large file preview, LLM layer-by-layer loading.

Section 5: LLM Weight Streaming

Inference without loading entire model into RAM.

FileRole
pogls_model_index.hMap layer → chunk list
pogls_weight_stream.hStream weights per layer

Target

  • Model 7B (14GB) → Actual RAM usage < 2GB
  • Latency per layer → < 100ms (NVMe SSD)

Frozen Rules

  • PHI constants from pogls_platform.h only
  • DiamondBlock 64B format - no changes
  • QRPN verify every chunk before output
  • GPU - do not modify commit path
  • Integer only - no float
  • core_c/ - do not modify

Success Criteria

  • Full Reconstruction: Text/code/weights files compress and reconstruct 100%
  • Partial Streaming: Chunk 1 available before chunk N arrives
  • LLM Inference: 7B model runs with < 2GB RAM
  • No Regression: All existing tests pass (2000+ cases)
