Key Findings
F-01

PHI Constants Dual-Define Conflict ใน pogls_delta.h

HIGH

pogls_delta.h define PHI_UP=1697054 และ PHI_DOWN=648609 แต่ pogls_fold.h และ pogls_fibo_addr.h ใช้ 1696631 / 648055 ซึ่งเป็นค่าที่ S19 ยึดถือ

  • ถ้า delta.h ถูก include ก่อน fold.h → PHI_UP ผิด → address calculation drift
  • delta lane block ที่เขียนด้วยค่าผิดจะ reconstruct ไม่ตรง
  • ควรลบ PHI defines ออกจาก delta.h — ใช้จาก platform.h อย่างเดียว
ลบ PHI_UP/PHI_DOWN/PHI_SCALE จาก pogls_delta.h → include pogls_platform.h แทน
F-02

pogls_audit.h — anomaly_flags ไม่มีใน PoglsStatsOut S19

HIGH

pogls_audit.h มี anomaly bitmap 6 ประเภทที่ละเอียดกว่า:

  • ANOMALY_HASH_MISMATCH (0x01), ANOMALY_COORD_DRIFT (0x02)
  • ANOMALY_OVERLAP_DELTA (0x04), ANOMALY_WARP_CORRUPT (0x08)
  • ANOMALY_DEEP_UNREADABLE (0x10), ANOMALY_SEQUENCE_BREAK (0x20)

S19 PoglsStatsOut มี 19 fields แต่ไม่มี anomaly classification — /llm_report จะรายงานได้แค่ stats ตัวเลข ไม่รู้ว่า "coord drift" หรือ "sequence break"

Port anomaly_flags (uint8_t) เข้า PoglsStatsOut เป็น field 20 — เพิ่ม version sentinel
F-03

pogls_5world_core.h — Orphaned แต่ Logic ตรงกับ ThirdEye

ORPHANED

P5WorldState มี address 5 แบบพร้อมกัน (icosa/dodeca/fibo + neg variants) พร้อม P5WorldCheck.aggregate_ok ที่ XOR ทั้ง 5 โลก = 0 — เป็น quantum parity invariant

  • ThirdEye มี 3 layers (NORMAL/STRESSED/ANOMALY) — 5World มี 5 variants
  • p5world_dual_map() ใช้ bit-rotate 10 — ตรงกับ Rubik permutation pattern
  • ไม่มี reference ใน geo_config.h หรือ geo_pipeline_v5.h เลย
Document ว่า 5World concept merge เข้า ThirdEye หรือ intentionally dropped — ถ้า keep: wire เข้า geo_hardening.h เป็น verify layer
F-04

pogls_mesh.h + pogls_mesh_entry.h — Voronoi/Delaunay Layer ไม่ถูก Port

DROPPED

Mesh layer เป็น async observer นอก hot path รับ DetachEntry → cluster via Voronoi (9 seeds × 3 slices = 27) → Delaunay edge graph

  • 9 seeds × 3 slices = 27 = 162/6 — ตรงกับ sacred number alignment
  • ReflexBias ใน mesh_entry.h คือ short-term muscle memory — POGLS38 planned แต่ไม่ถึง
  • depends on pogls_detach_lane.h และ pogls_engine_slice.h ซึ่งไม่มีใน zip
Mark as FUTURE — concept ดีแต่ dependencies missing, ไม่ควร port ก่อน detach_lane เสร็จ
F-05

pogls_frontier.h — BFS Logic ซ้ำกับ geo_net.h แต่ Missing Dependency

DROPPED

pogls_frontier.h มี frontier_expand() BFS radius-N propagation บน bitboard graph แต่ depend on pogls_graph_topology.h ที่ไม่มีในทั้ง S10 zip และ S19 context

  • frontier_new_only() = BFS visited masking — logic ตรงกับ QRPN traversal ใน geo_net.h
  • frontier_parity() = bb_popcount mod 2 — ใช้เป็น integrity fingerprint ได้
Extract frontier_parity() เข้า geo_whe.h เป็น parity check primitive — ส่วน frontier_expand ทิ้งไว้ก่อน
F-06

pogls_pipeline_wire.h — S10 Version ยังถูกต้อง Architecturally

SUPERSEDED (OK)

S10 pipeline_wire มีโครงสร้าง [GeoNet+RH] → [Geomatrix FILTER] → [GeoNet VERIFY] + QRPN check ครบถ้วน — S19 geo_pipeline_v5.h แค่ optimize ให้เร็วขึ้น (530 M/s) ไม่ได้เปลี่ยน logic

ข้อสังเกต: S10 pipeline_wire.h depend on geo_config.h ซึ่งยังไม่มีตอน S10 — แปลว่า file นี้ถูกเขียนแบบ forward-compatible รอ geo_config.h
